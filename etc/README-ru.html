FinMiner от FINOM

версия: 2.2

FinMiner — программный продукт, разработанный компанией Finom, для создания структурных криптовалютных единиц на основе алгоритма Ethash. Настоящая версия FinMiner создана для работы со всеми криптовалютами, основанными на указанном алгоритме, такими как Ethereum, Ethereum Classic и другими. Данная версия FinMiner работает в операционных системах Windows или Linux с видеокартами AMD и Nvidia.

Для того, чтобы начать добывать Ethereum с помощью FinMiner, в конфигурационном файле достаточно указать только Ваш кошелек.

Испытания FinMiner показали высокую эффективность при работе с Ethereum и Ethereum Classic. По результатам проведенных исследований, производительность FinMiner в целом не уступает, а часто даже превосходит производительность конкурирующих программных продуктов. При этом FinMiner отличается высокой стабильностью и простотой настройки.

Оплата

Оплата пользования FinMiner осуществляется посредством комиссионного майнинга. Комиссия составляет 1% от общего времени майнинга: в произвольный момент каждого часа работы FinMiner на 36 секунд переключает майнинг на кошельки, принадлежащие компании Finom.

Настройка

При запуске FinMiner читает файл настроек config.ini из текущего каталога программы. При запуске с ключом -d (Пример: finminer.exe -d) выводится список устройств, обнаруженных майнером, их PCI-адреса и размер памяти. Для использования данной функции в OS Windows необходимо запустить программу из консоли (cmd).

FinMiner не требует указания пулов в конфигурационном файле. Если пул (список пулов) не указан, FinMiner в автоматическом режиме будет использовать пулы nanopool.org, соответствующие выбранной криптовалюте.

При старте FinMiner выводит в консольный лог основную рабочую информацию, включающую текущую версию программы, имя рига, количество и тип установленных видеокарт, а также сведения о текущих настройках программы.

Log-файлы

Функция логирования событий в FinMiner запускается автоматически при старте программы. Создаваемые log-файлы содержат всю информацию, отображаемую в консоли при работе майнера. По умолчанию log-файлы складываются в папку logs текущего каталога программы. Отключить логирование, а также задать произвольный каталог для записи log-файлов можно использовав соответствующие параметры конфигурирования (смотри примеры в разделе Параметры настоящего руководства).

Удалённый мониторинг

FinMiner поддерживает сетевой API программы EthMan для мониторинга ригов. По умолчанию открывается порт 3333 в режиме “только для чтения” без возможности перезапуска майнера или рига по сети. В конфигурационном файле программы можно настроить порт либо отключить функцию API опцией mport, там же можно задать пароль для мониторинга опцией ethmanPassword.

Функция автоматического перезапуска майнера

FinMiner c настройками по умолчанию при возникновении критических ошибок на GPU либо их зависании (обычно подобные ошибки возникают при наличие аппаратных проблемы или из-за завышения частот функционирования GPU – “переразгона”) сам перезапускается. Функцию автоматического перезапуска можно отключить используя параметр watchdog.

Также, параметр minHashrate (минимальный хешрэйт) позволяет задать значение минимального хешрэйта при котором будет произведён перезапуск майнера. Для работы данная функция использует усреднённое за 10 минут значение хешрэйта, отображаемое в консольном логе синим цветом. Если общий хешрэйт за 10 минут будет меньше заданного параметра то майнер перезапустится. По умолчанию порог для минимального хешрэйта не задан.

Ещё одна функция FinMiner, способствующая улучшению автономной работы майнера, это функция задаваемая параметром restarts. Она задаёт количество перезапусков майнера перед перезапуском рабочей машины (рига). По умолчанию майнер всегда будет перезапускать только сам себя.
Подробную информацию об использовании указанных функций можно найти в разделе Параметры настоящего руководства.

Параметры

Настройки FinMiner находятся в конфигурационном файле config.ini в так называемом INI-формате в виде логических пар параметр=значение. В данном файле допускаются пустые строки и комментарии. Строки с комментариями должны начинаться со знака “;” (точка с запятой). Параметры и значения не учитывают регистр ввода, то есть для программы нет разницы между ETH, eth, Eth или между wallet и Wallet. Если для какого-то параметра указано некорректное значение, то вместо него подставляется значение по умолчанию (данное правило не работает для параметра wallet).

Ниже приведен список параметров, которые можно задать в FinMiner.

wallet

Обязательный параметр.
Кошелек пользователя, на который будут переводиться средства.

coin

Условно необязательный параметр.
Выбор монеты для пула по умолчанию. Пулом по умолчанию является nanopool.org.
Параметр coin может принимать одно из двух значений: ETH (для Ethereum) и ETC (для Ethereum Classic). При указании монеты FinMiner сам определит необходимые для работы пулы, если они не заданы отдельным параметром. В случае, если монета не задана, FinMiner использует монету по умолчанию – Ethereum. Кроме того, если в конфигурационном файле указан nanopool.org для Ethereum или Ethereum Classic, то FinMiner может определить монету из настроек пула.

Важно: при использовании FinMiner для получения Ethereum Classic на пуле по умолчанию следует указать монету (coin=ETC), в этом случае пулы определятся автоматически.

Если пулы заданы явно при помощи параметров pool1, pool2, …, то FinMiner будет работать согласно заданиям, приходящим с пулов.

rigName

Необязательный параметр.
Имя рига (компьютера/рабочей машины). Введённый параметр будет отображаться в статистике пула. Если этот параметр не задан, программа сгенерирует уникальное имя и передаст на пул.

email

Необязательный параметр.
Электронная почта пользователя. Данный параметр передается на пул, в котором будет работать риг. Пул может использовать ее для рассылки сервисных сообщений и уведомлений.

pool1, pool2, …

Необязательные параметры.
Пулы для майнинга. Значения в виде url:port (например: pool1=eth-eu1.nanopool.org:9999). Параметры должны быть указаны в порядке возрастания, с последовательной нумерацией от pool1 до poolN (например: pool1, pool2, pool3). Пул будет выбран из списка автоматически, в соответствии с максимальной скоростью соединения. Если пул (список пулов) не указан, FinMiner в автоматическом режиме будет использовать пулы nanopool.org, соответствующие выбранной криптовалюте.

rigPassword

Необязательный параметр.
Пароль для рига (рабочей машины). Может потребоваться для работы с пулами, где есть регистрация и задан пароль для рига.

watchdog

Необязательный параметр.
Данный параметр управляет функцией перезапуска майнера при серьёзных ошибках на GPU либо их зависании. Может принимать значения true и false, по умолчанию true – автоматический перезапуск включён.

minHashrate

Необязательный параметр.
Минимально допустимый хешрэйт. Функция следит за общим хешрэйтом рига и сравнивает его с заданным параметром. Если в течение пяти минут после запуска майнера заданный предел не достигнут то FinMiner автоматически перезапускается. Так же, перезапуск майнера будет произведён, если усреднённое за 10 минут значение хешрэйта по каким-то причинам опустится ниже заданного предела. Значение это вещественное число с опциональной буквой-модификатором, обозначающей тысячу для килохешэй или миллион для мегахешэй в секунду. Например, порог в 100 мегахешэй в секунду можно записать как 100M, 100.0M, 100m, 100000k, 100000K или 100000000. Если параметр не указан, то майнер не будет перезапускаться (за исключением случаев, описанных в разделе watchdog).

devices

Необязательный параметр.
Видеокарты, которые будет использовать майнер. При необходимости запустить майнер не на всех доступных GPU, а только на некоторых, их номера можно передать в параметре devices через запятую. FinMiner нумерует GPU начиная с нуля, в порядке возрастания PCI-адреса. Увидеть список доступных GPU и их порядок можно запустив FinMiner с ключом -d:

finminer -d

Например, если всего в системе четыре GPU (0, 1, 2, 3) а майнить необходимо на всех кроме предпоследней (с индексом 2) то необходимо задать опцию devices следующим образом:

devices=0,1,3

Порядок устройств определяет порядок отображения хешрэйта. Например, при указании

devices=3,1,0

в строке с хешрэйтом будет отображаться сначала GPU3, затем GPU1 и GPU0.

restarts

Необязательный параметр.
Данный параметр задаёт количество перезапусков майнера перед попыткой перезагрузки рига. При возникновении таких проблем на GPU, как аппаратные ошибки или зависания, а также при деградации хешрэйта (если задана опция minhashrate) FinMiner перезапускается. Однако некоторые ошибки невозможно исправить перезапуском процесса и необходимо перезагружать риг. Параметр restarts определяет количество перезапусков, которое производит майнер перед тем как попытаться перезагрузить всю систему. Для перезагрузки майнер запускает из текущего каталога скрипт reboot.bat под Windows или reboot.sh под Linux.
Типичное содержание скрипта reboot.sh под Linux:

reboot

Типичное содержание скрипта reboot.bat под Windows:

shutdown /r /t 5 /f

Скрипт требуется написать самостоятельно.

noLog

Необязательный параметр.
Может принимать значения true и false (по умолчанию false). При выставлении этого параметра в true log-файлы не пишутся на диск.

logDir

Необязательный параметр.
Имя папки для создания файлов с логами. Значение по умолчанию: logs/.

mport

Необязательный параметр.
Сетевой порт для удалённого мониторинга и управления программой EthMan или другими программами, использующими подобный формат протокола API. Поддерживается весь API, в том числе, перезапуск майнера и рига.
Поддерживается режим запрета управления майнером через API (в этом случе майнер показывает только статистику и не реагирует на команды). Для этого перед номером порта требуется поставить знак “минус”.
Поддерживается режим полного отключения сетевого мониторинга. Для этого требуется в качестве номера порта указать “0” (ноль).
Значение по умолчанию: -3333 (майнер работает в режиме запрета управления по API и показывает статистику на порту 3333).

ethmanPassword

Необязательный параметр.
Пароль для мониторинга с помощью EthMan и другими утилитами, поддерживающими тот же сетевой API.

Конфигурационный файл

Минимальный конфигурационный файл Ethereum может содержать только кошелёк:

wallet=<кошелёк>

FinMiner автоматически подставит пулы для Ethereum.

Чтобы работать с Ethereum Classic, следует добавить указание монеты:

wallet=<кошелёк>
coin=ETC

В этом случае FinMiner будет использовать пулы, соответствующие Ethereum Classic.

ВАЖНО!

Для монет на алгоритмах Ethash, не поддерживаемых nanopool.org, в обязательном порядке следует указать кошелек (wallet), и пулы (pool1…). В этом случае FinMiner будет функционировать по принципу работы Ethereum, но результаты операций будут определяться заданиями, приходящими с соответствующих пулов, которые указаны в файле конфигурации.

Примеры конфигурационных файлов

Пример файла конфигурации для Ethereum:

wallet = 0xffffffffffffffffffffffffffffffffffffffff
rigName = rig1
email = someemail@org
pool1 = eth-eu1.nanopool.org:9999
pool2 = eth-eu2.nanopool.org:9999
pool3 = eth-us-east1.nanopool.org:9999
pool4 = eth-us-west1.nanopool.org:9999
pool5 = eth-asia1.nanopool.org:9999
pool6 = eth-jp1.nanopool.org:9999
pool7 = eth-au1.nanopool.org:9999

Пример эквивалентного файла для Ethereum:

wallet = 0xffffffffffffffffffffffffffffffffffffffff
rigName = rig1
email = someemail@org

Пример минимального файла для Ethereum:

wallet=0xffffffffffffffffffffffffffffffffffffffff

Пример файла конфигурации для Ethereum Classic:

wallet = 0xffffffffffffffffffffffffffffffffffffffff
coin=Etc
rigName = rig1
email = someemail@org
pool1 = etc-eu1.nanopool.org:19999
pool2 = etc-eu2.nanopool.org:19999
pool3 = etc-us-east1.nanopool.org:19999
pool4 = etc-us-west1.nanopool.org:19999
pool5 = etc-asia1.nanopool.org:19999
pool6 = etc-jp1.nanopool.org:19999
pool7 = etc-au1.nanopool.org:19999

Пример эквивалентного файла для Ethereum Classic:

wallet = 0xffffffffffffffffffffffffffffffffffffffff
coin=Etc
rigName = rig1
email = someemail@org

Пример минимального файла для Ethereum Classic:

wallet=0xffffffffffffffffffffffffffffffffffffffff
coin=Etc
