FinMiner de FINOM

versión 2.2

FinMiner es un programa-producto desarrollado por la compañía Finom para crear unidades estructurales de criptomoneda que utilizan el algoritmo Ethash. La versión actual de FinMiner fue creada para manejar todas las criptomonedas basadas en ese algoritmo: Ethereum, Ethereum Classic y varias más. Esta versión de FinMiner puede ser usada con Windows o Linux y con tarjetas de video AMD o Nvidia.

Para poder empezar a trabajar con FinMiner, basta sólo con ingresar el número de tu monedero en el archivo de configuración.

Las pruebas que se han hecho con FinMiner muestran un alto rendimiento al trabajar con Ethereum y Ethereum Classic. Los resultados de la investigación demuestran que FinMiner trabaja a un nivel parejo o mayor que el de la competencia. FinMiner también se distingue por su alta estabilidad y simple configuración.

Pago

Pago por el uso de FinMiner toma la forma de una comisión de la minería. La comisión es 1% del tiempo total de minería: una vez cada hora al azar FinMiner desvía la minería a sus monederos por 36 segundos.

Configuración

Al abrirse, FinMiner lee el archivo config.ini del directorio actual del programa. Si se abre con la opción -d en la línea de comandos (por ejemplo, finminer.exe -d) el minador muestra una lista de los dispositivos que detecta, incluyendo sus direcciones PCI y su cantidad de memoria. Para utilizar esta función en Windows, es necesario abrir el programa en la línea de comandos (cmd).

Al usar FinMiner, no es necesario especificar los pools en el archivo de configuración. Si el pool (o lista de pools) no se especifica, FinMiner automáticamente utilizará los pools en nanopool.org para la criptomoneda escogida.

FinMiner al abrir muestra en el log (registro) de la consola la información principal de trabajo, que incluye la versión actual del programa, el nombre del rig, la cantidad y tipo de tarjetas de video instaladas y las configuraciones actuales del programa.

Archivos log

La función de crear logs de eventos en FinMiner se activa automáticamente cada vez que se abre el programa. Los archivos log creados contienen toda la información que se muestra en la consola cuando el minador está en operación. Por defecto, los archivos log se guardan en la carpeta log en el directorio actual del programa. Se puede desactivar el logueo de eventos o asignar un catálogo al azar para guardar los archivos log utilizando los parámetros de configuración correspondientes (ve los ejemplos en la sección Parámetros de este archivo).

Monitoreo remoto

FinMiner apoya el programa EthMan de API de red para monitoreo de rigs. Por defecto abre el puerto 3333 en modo de “sólo lectura” sin la abilidad de reiniciar el minador o rig a través de la red. En el archivo de configuración del programa, el puerto puede ser cambiado y la función API desactivada en el parámetro mport. El archivo de configuración también te permite definir una contraseña para monitoreo con la opción ethmanPassword.

Función de reinicio automático

Con la configuración por defecto, FinMiner automáticamente se reiniciará si se topa con errores críticos de GPU o lapsos en rendimiento. Por lo general estos errores surgen por problemas de equipo o por overclocking. La función de reinicio automático puede ser desactivada usando el parámetro watchdog.

De igual manera, el parámetro minHashrate (hashrate mínimo) permite que el usuario defina un nivel de hashrate mínimo. Si el nive actual cae debajo de este límite, el minador se reiniciará. Esta función toma en cuanta el hashrate promedio de los últimos 10 minutos, mostrado en azul en el log de la consola. En la configuración por defecto no está definido el hashrate mínimo.

Otra función de FinMiner que mejora el rendimiento automático del minador es el parámetro restarts. Asigna el número de veces que el minador se reiniciará antes de reiniciar el worker (rig). Por defecto el minador sólo se reiniciará a sí mismo.

Infomación más detallada sobre cómo utilizar estas funciones podrá encontrarse en la sección Parametros de este archivo.

Parámetros

La configuración de FinMiner se realiza en el archivo de configuración config.ini en la forma de pares de lógica parámetro=definición. Este archivo permite la presencia de líneas y comentarios vacíos. Las líneas de comentario deben de comenzar con un ; (punto y coma). Los parámetros y definiciones no son sensibles a las mayúsculas, lo cual significa que para el programa no hay ninguna diferencia si escribes ETH, eth o Eth, ni tampoco si escribes wallet o Wallet. Si se define un parámetro de manera incorrecta, se utilizará la definición por defecto (esta regla no aplica al parámetro wallet).

Lo que sigue es una lista de los parámetros que pueden ser configurados en FinMiner:

wallet

Parámetro obligatorio.
Este es el monedero del usuario, donde se depositarán fondos.

coin

Parámetro opcional.
Define la moneda que se minará por defecto para el pool. El pool por defecto es nanopool.org. El parámetro coin acepta una de dos definiciones: ETH (para Ethereum) y ETC (para Ethereum Classic). Cuando se especifica una moneda, FinMiner automáticamente determina el pool necesario para que funcione si no se ha proporcionado ninguno en un parámetro adicional. Si no se especifica una moneda, FinMiner utilizará Ethereum, la moneda por defecto. Y si nanopool.org se ha especificado en el archivo de configuración para Ethereum o Ethereum Classic, FinMiner puede determinar la moneda de la configuración del pool.

Importante: al usar FinMiner para minar Ethereum Classic en el pool dado por defecto, es necesario definir la moneda (coin=ETC). En ese caso los pools serán determinados automáticamente.

Si los pools están cláramente definidos con la ayuda del parámetro pool1, pool2, …, FinMiner funcionará según las tareas que recibe de esos pools.

rigName

Parámetro opcional.
Consiste en el nombre del rig (computadora o worker). Muestra las estadísticas del pool. Si este parámetro no se defina, el programa generará un nombre único para el rig y lo pasará al pool.

email

Parámetro opcional.
Consiste en la dirección de correo electrónico del usuario. Se proporciona al pool donde estará operando el rig. El pool lo puede usar para mandar notificaciones de servicio.

pool1, pool2, …

Parámetro opcional.
Consiste en los pools de minería utilizados. Las definiciones deben de estar en el formato url:port (por ejemplo: “pool1=eth-eu1.nanopool.org:9999”). Los parámetros deben ser definidos en orden ascendente y secuencial, de pool1 a poolN (por ejemplo: pool1, pool2, pool3). El pool será seleccionado automáticamente de la lista según la conexión más rápida. Si el pool (o lista de pools) no se define, FinMiner automáticamente utilizará los pools en nanopool.org que corresponden a esa criptomoneda.

rigPassword

Parámetro opcional.
La contraseña para el rig (o worker). Podría ser necesario utilizarla al trabajar con pools que requieren registración y una contraseña para el rig.

watchdog

Parámetro opcional.
Este parámetro maneja la función de reinicio cuando el programa se topa con errores críticos de GPU o lapsos en rendimiento. Acepta las definiciones true o false. Por defecto, true – reinicio automático – está activado.

minHashrate

Parámetro opcional.
Este parámetro define el nivel mínimo de hashrate aceptable. Lleva un registro del hashrate total del rig y lo compara con el nivel definido en este parámetro. Si cinco minutos después de iniciarse no se ha logrado alcanzar el nivel mínimo, FinMiner se reiniciara automáticamente. De igual manera, el minador se reiniciará si por cualquier razón el hashrate promedio de los últimos diez minutos cae debajo del nivel definido. Esta definición puede llevar una letra modificadora opcional que representa mil para kilohash o un millón para megahash por segundo. Por ejemplo, asignar 100 megahashes por segundo puede ser escrito 100M, 100.0M, 100m, 100000k, 100000K o 100000000. Si este parámetro no se define, el minador no se reiniciará (con la excepción de la situación descrita en la sección watchdog section).

devices

Parámetro opcional.
Estas son las tarjetas de video que el minador utilizará. Si queres que el minador utilice sólo algunas de las tarjetas disponibles, sus números pueden ser definidos en el parámetro devices separados por una coma. FinMiner enumera los GPUs comenzando desde cero en orden ascendente de su dirección PCI. Puedes ver una lista de GPUs disponibles y el orden en el que están iniciando FinMiner con la opción -d en la línea de comandos:

finminer -d

Por ejemplo, si hay cuatro GPUs en el sistema (0, 1, 2, 3) y todas menos la penúltima (indicada con el número 2) deben de ser asignadas para minar, la opción devices debe de ser asignada de la siguiente manera:

devices=0,1,3

El orden de dispositivos determina el orden de hashrate mostrado. Por ejemplo, si se define como

devices=3,1,0

la línea hashrate mostrará primero GPU3, luego GPU1 y finalmente GPU0.

restarts

Parámetro opcional.
Este parámetro asigna el número de veces que el minador se reiniciará antes de reiniciar el rig. En caso de problemas de GPU como errores de equipo o lapsos en rendimiento, o en caso de degradación de hashrate (si se usa la opción minhashrate), FinMiner se reiniciará. Sin embargo, ciertos errores no pueden ser resueltos reinicando el programa. En estos casos es necesario reiniciar el rig. Para reiniciar, el minador carga el script reboot.bat del directorio actual en Windows o reboot.sh en Linux:

reboot

El típico contenido del script reboot.bat para Windows:

shutdown /r /t 5 /f

El script tiene que ser escrito por el usuario.

noLog

Parámetro opcional.
Este parámetro acepta las definiciones true o false (por defecto false). Si este parámetro se define como true, no se guardarán archivos log en el disco duro.

logDir

Parámetro opcional.
Este es el nombre de la carpeta en la cual se crearán archivos log. La carpeta de destino por defecto es logs/.

mport

Parámetro opcional.
Este es el puerto de red para monitoreo remoto y manejo del programa a través de EthMan u otros programas que utilizan un formato de protocolo API parecido.
El programa apoya todas las funciones de API, incluyendo el reinicio del minador y rig(s).
Puedes bloquear el manejo del minador a través de API (en cuyo caso el minador sólo mostrará las estadísticas y no responderá a comandos). Para activar esta función, un signo menos (-) debe de ser escrito antes del número de puerto.
Y también puedes completamente desactivar monitoreo remoto. Para hacer esto, hay que definir el número de puerto como “0” (cero).
Definición por defecto: -3333. (Esto significa que el minador bloquea manejo a través de API y muestra las estadísticas en puerto 3333).

ethmanPassword

Parámetro opcional.
Tu contraseña para monitorear con EthMan y otras utilidades que utilizan el mismo API de red.

Archivo de configuración

El archivo de configuración para Ethereum puede consistir sólo en un monedero:

wallet=<wallet>

FinMiner automáticamente utilizará pools para Ethereum.

Para trabajar con Ethereum Classic, la moneda debe de ser especificada:

wallet=<wallet>
coin=ETC

En este caso FinMiner utilizará los pools que le corresponden a Ethereum Classic.

¡IMPORTANTE!

Para monedas que funcionan con el algoritmo Ethash pero que no están apoyados por nanopool.org, tienes que especificar un wallet y pools (pool1…). En este caso FinMiner funcionará como si estuviera trabajando con Ethereum, pero los resultados de las operaciones se determinarán por los pools en el archivo de configuración.

Ejemplos de archivos de configuración

Un ejemplo de un archivo de configuración para Ethereum:

wallet = 0xffffffffffffffffffffffffffffffffffffffff
rigName = rig1
email = someemail@org
pool1 = eth-eu1.nanopool.org:9999
pool2 = eth-eu2.nanopool.org:9999
pool3 = eth-us-east1.nanopool.org:9999
pool4 = eth-us-west1.nanopool.org:9999
pool5 = eth-asia1.nanopool.org:9999
pool6 = eth-jp1.nanopool.org:9999
pool7 = eth-au1.nanopool.org:9999

Un ejemplo de un archivo equivalente para Ethereum:

wallet = 0xffffffffffffffffffffffffffffffffffffffff
rigName = rig1
email = someemail@org

Un ejemplo de un archivo mínimo para Ethereum:

wallet=0xffffffffffffffffffffffffffffffffffffffff

Un ejemplo de un archivo completo para Ethereum Classic:

wallet = 0xffffffffffffffffffffffffffffffffffffffff
coin=Etc
rigName = rig1
email = someemail@org
pool1 = etc-eu1.nanopool.org:19999
pool2 = etc-eu2.nanopool.org:19999
pool3 = etc-us-east1.nanopool.org:19999
pool4 = etc-us-west1.nanopool.org:19999
pool5 = etc-asia1.nanopool.org:19999
pool6 = etc-jp1.nanopool.org:19999
pool7 = etc-au1.nanopool.org:19999

Un ejemplo de un archivo equivalente para Ethereum Classic:

wallet = 0xffffffffffffffffffffffffffffffffffffffff
coin=Etc
rigName = rig1
email = someemail@org

Un ejemplo de un archivo mínimo para Ethereum Classic:

wallet=0xffffffffffffffffffffffffffffffffffffffff
coin=Etc
